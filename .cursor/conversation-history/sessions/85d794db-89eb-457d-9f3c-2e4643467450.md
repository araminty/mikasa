Yes, on many screens — with caveats.

**How it works:** CSS sets the *layout* size (120×150 CSS pixels). The image file has its own pixel count. If the file is **240×300** but still shown in a 120×150 box, the browser **shrinks** it. On a **2×** display (Retina, most laptops, many phones), that often looks **sharper** than a 120×150 file shown at the same size, because you’re feeding more real pixels into each CSS pixel.

**When it helps**
- Retina / HiDPI monitors and phones  
- Photos and detailed art (less mushy when scaled down)

**When it barely matters**
- Plain 1× displays (1:1 asset is already fine)  
- Very simple flat graphics with hard edges (PNG at 1× can already look crisp)

**Tradeoffs**
- **Larger files** (roughly up to ~4× the pixels, though compression helps)  
- Slightly more memory in the browser — usually negligible for a handful of card images

**Practical rule:** Export **240×300** (2×) for card art, reference it the same way in `card-image: img/foo.webp`, and keep CSS at 120×150. No CSS change required.

**Won’t fix:** A blurry source, heavy JPEG compression, or bad cropping from `object-fit: cover` — 2× can’t invent detail that isn’t there.

So: **yes for quality on modern displays; optional but a good habit for card portraits.**
